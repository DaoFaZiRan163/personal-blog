核心技术设计决策
核心创新 LLM 五层 Harness 体系 把"接入 LLM"拆成五个独立工程层：Eval（质量评估）→ Multi-Model Router（成本/能力路由）→ 本地推理（Ollama / MLC LLM 边缘部署）→ Guardrail（输出护栏）→ Observability（Langfuse 全链路追踪），每层均以生产 SLA 为目标设计。 成本控制 多模型动态路由 根据查询复杂度、数据敏感度、单次成本预算三维评分，动态路由至 Claude API（高复杂度）/ Ollama 本地（低敏感度常规查询）/ MLC LLM 边缘（离线场景），LLM 调用成本降低 42%，58% 请求由本地模型处理、零 API 费用。 质量门控 Eval-as-CI-gate 用 Inspect AI 构建覆盖意图识别、数值计算、幻觉检测三类场景的评估套件，接入 GitHub Actions，每个 PR 自动触发 eval 运行；Eval 分数低于阈值则阻断合并，确保 LLM 行为变更有质量保障。 AI 集成 MCP Server 暴露核心引擎 为核心计算引擎设计并实现 MCP Server 接口，AI 可直接调用"查询本月收益"、"预测下月收入"等工具，实现真正 AI-native 的数据交互，无需用户手动导出报表。 安全加固 Prompt Injection 红队测试 对所有 LLM 入口（用户问题输入、Mailgun 邮件解析、MCP 工具调用结果）系统性实施红队测试，发现并修复 3 类注入路径；同步接入 Langfuse 实时 trace，异常 prompt 模式 30 秒内可告警。 HITL 设计 长任务 + 人工介入节点 异常申诉、高风险核查等场景采用 HITL（Human-in-the-Loop）设计：AI 完成初步分析 → 挂起等待人工确认 → 确认后继续执行，通过 Hangfire 后台任务 + WebSocket 实时推送状态，保证长任务可中断可追踪。

核心产品设计决策

Project 02 · 2025

宝可梦卡牌 Web3 交易平台

核心产品决策
用户痛点 实体卡牌三大核心痛点 日本卡牌玩家面临：①真假难辨——二手市场假卡泛滥，PSA 评级门槛高；②价格不透明——Yahoo 拍卖与 Mercari 成交分散，无统一估值参考；③流动性差——高价值卡牌难以快速变现，卖家需长期等待买家。三大痛点共同指向"可信确权 + 实时定价 + 即时流动性"的核心产品方向。 竞品驱动 对标 CollectorCrypt 差异化设计 深度分析竞品 CollectorCrypt（市值 1,250 万美元），提炼三项借鉴点：即时回购（85% 市价）、透明概率盲盒、社区文化建设；同时识别对方手续费机制不透明的缺陷，设计差异化的阶梯手续费体系（1.5% 普通 / 0.5% 早期用户 / VIP 折扣）。 商业创新 回购 → 透明盲盒 业务闭环 设计"平台回购 PSA10 卡牌（市价 85-90%）→ 重新组成公示概率盲盒销售"的业务闭环，既消化库存、又产生额外收入；同步完成日本《景品表示法》合规分析，确保概率公示满足监管要求。 数据驱动 实时价格抓取驱动定价与收藏估值 设计 Python 爬虫系统每日抓取 Yahoo 拍卖、Mercari 成交数据，构建卡牌历史价格库；用户收藏总价值实时联动最新市场价计算，价格数据结构预留链上价格预言机接入能力。 战略决策 Web2 先行、Web3 平滑迁移 放弃"一步到位 Web3"方案，先以 Web2 MVP（2-3 个月）快速验证市场需求、积累价格数据与用户行为，规避早期合规风险；数据结构设计完全兼容链上格式，确保后续 1:1 迁移至 Polygon。 合规研究 日本三部法律合规边界分析 系统梳理《景品表示法》（概率公示要求）、《コンプリートガチャ 禁令》（完整扭蛋禁止）、《古物営業法》（二手商品交易规范），逐条对应产品功能设计，明确卡包盲盒、回购机制各自的合规路径与禁区。
技术栈
Next.js / React Node.js + TypeScript PostgreSQL Redis Polygon / ERC-721 Python Scrapy / Playwright Vercel + Railway JWT
我的角色（Forward Deployed 视角）
产品设计：主导从竞品分析到 PRD 的全流程，输出三阶段产品路线图，明确 Web2 MVP → Web3 迁移的里程碑
合规研究：系统梳理日本三部法律（景品表示法、扭蛋禁令、古物営業法），逐条对应产品功能，划定合规边界
技术方案：设计链上确权架构（Polygon ERC-721）与爬虫定价系统，数据结构完全兼容后续链上迁移

核心架构设计决策
架构 三端 + 七服务微服务拆分 Mall 会员端、Org 分公司端、Hq 总部后台三端独立部署；后端按业务域拆分为 BaseServiceCenter（基础服务）、ServiceCenterMall / Hq / Org / Search / Usdt / ImgService 七大服务，各自独立迭代，互不干扰。 核心设计 自研分库路由算法 基于 CustomerID / OrderID / CustomerNO / Mobile 等多维路由键，通过数据中心编号奇偶分片，实现多数据库无感知路由，支撑大规模会员数据水平扩展。 分层缓存 Redis 业务对象缓存体系 针对商品分类、品牌、公告、广告、字典等高频只读数据，设计 20+ 专项 Cache 类，结合 Redis 实现多级缓存，大幅降低数据库压力。 国际化 多语言 + 多货币实时汇率 支持中文、英文、日文、越南文、印尼文、繁体中文 6 种语言动态切换；接入第三方汇率 API，每 30 分钟自动刷新 MYR、HKD、USD、CNY 等货币对，管理端可手动触发。 支付创新 USDT 加密支付双链集成 独立 UsdtService 服务，同时支持 TRC-20（Tron）和 ERC-20（Ethereum）链上 USDT 收款与转账，内置 Gas 动态估算、测试网 / 主网一键切换、交易记录持久化。
